## âš¡ QUICK REFERENCE â€” THE ESSENTIAL RULES

If you read nothing else, these are the most important rules:

### The 10 Most Critical Programming Rules

```
1. CLARITY OVER CLEVERNESS
   Write code for the human who reads it, not just the machine that runs it.

2. FAIL FAST AND LOUDLY
   Validate inputs at boundaries. Never silently swallow errors.

3. ONE THING, ONE PLACE
   Functions do one thing. Classes have one responsibility. No duplication.

4. NAMES ARE DOCUMENTATION
   A good name eliminates the need for a comment. Invest in naming.

5. SECURITY IS NOT OPTIONAL
   No hardcoded secrets. No string-concatenated SQL. Always validate input.

6. TEST BEHAVIOR, NOT IMPLEMENTATION
   Tests must survive refactoring. They must catch bugs when they happen.

7. COMPLEXITY IS THE ENEMY
   The simplest solution that correctly solves the problem is the best one.

8. MEASURE BEFORE OPTIMIZING
   Never optimize without profiling data. Premature optimization causes bugs.

9. GIT IS YOUR SAFETY NET
   Small, focused commits with meaningful messages. One concern per commit.

10. PRODUCTION IS THE STANDARD
    Every line of code you write should be ready to run in production now.
```
